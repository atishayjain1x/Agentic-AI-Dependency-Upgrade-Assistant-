# Java 11 to Java 17 Migration Notes

## Overview

Java 17 is a Long-Term Support (LTS) release and serves as the baseline for many modern Java frameworks including Spring Boot 3, Hibernate 6, and Micronaut 4.

Benefits include:

* Long-term support
* Improved performance
* Enhanced security
* Modern language features
* Better JVM diagnostics

## Prerequisites

Before upgrading:

* Upgrade build servers to Java 17
* Upgrade local developer environments
* Verify CI/CD pipelines
* Verify container base images

## Language Features

### Records

Before:

```java
public class User {
    private final String name;

    public User(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
```

After:

```java
public record User(String name) {
}
```

### Pattern Matching for instanceof

Before:

```java
if (obj instanceof String) {
    String s = (String) obj;
}
```

After:

```java
if (obj instanceof String s) {
}
```

### Sealed Classes

```java
public sealed interface Shape
    permits Circle, Rectangle {
}
```

## Strong Encapsulation

Java 17 strongly encapsulates internal JDK APIs.

Review usage of:

```java
sun.misc.*
com.sun.*
jdk.internal.*
```

Replace internal APIs with supported alternatives.

## Removed Components

### Nashorn JavaScript Engine

Review:

* ScriptEngineManager usage
* Embedded JavaScript execution

### Pack200

Review:

* Packaging pipelines
* Artifact compression workflows

## Reflection and Instrumentation

Investigate:

* Illegal reflective access warnings
* Bytecode enhancement libraries
* Reflection-heavy frameworks

## Testing Recommendations

Verify:

* Serialization
* Reflection
* Security modules
* Integration tests
* Build plugins

## Upgrade Checklist

1. Upgrade JDK.
2. Upgrade CI infrastructure.
3. Remove internal API dependencies.
4. Resolve compiler warnings.
5. Execute regression tests.
6. Validate production workloads.
